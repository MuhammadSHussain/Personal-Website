Webpage structure

The html files include three sections from a header, body and footer.

Header: The header is a fixed element at the top of every page that contains the navigation menu and title. It helps users to navigate the website. The header is consistent across all pages, allowing users to browse the site more easily.

Main/ Body: The primary content part on each page contains all relevant information about the home, past, future, and comments pages. It incorporates text, photographs, and tables to provide context and information. Images augment the text and offer visual appeal, whilst tables organise and display information in a formatted manner.

Footer: The footer, which appears at the bottom of each page, includes basic information about the author, organization, year, and author identification. It is consistent throughout all pages, creating a cohesive ending to the website. The footer may also provide further links, contact information, or copyright information.

Aesthetics Aspects

The websystems CSS file determines the visuals of the website, such as layout, colours, fonts, spacing, and structure. It defines the site's overall appearance and feel, through various CSS selectors that target elements are based on the tag name such as body and header, classes like .h1, .Grey, etc. ID which is #footer, also lastly pseudo-classes for instance :hover, :nth-child(even).

Images: Images contribute significantly to the aesthetics of a website, offering visual appeal as well as context for the material. They should be of high quality, relevant, and well-optimized for online use to ensure fast loading times. Using photographs that complement the overall design and theme of the website results in a more engaging user experience.

Colour and Fonts: The colour palette chosen is Grey, Verdigris and Cerulean (#9F9F9F, #70C1B3, #247B9F) for background gradient and a darker version of Cerulean for the headers and footer to create a cohesive visual experience. Considerations should be made for accessibility, ensuring that text is easily readable against the background colour. Fonts should be easy to read and appropriate for the content and audience. Using a combination of fonts for headings and body text can help create hierarchy and visual interest. Web fonts can be used to ensure consistency across different devices and browsers.

Structure and Interactivity: A well-structured website has clear hierarchy and logical flow. This includes the organizing of objects like as headlines, paragraphs, images, and navigation menus. Consistent spacing and alignment help to create a professional image. Interactivity enhances the user experience by allowing feedback. Hover effects, animations, sliders, and interactive forms may all be included. Interactive elements should be used selectively to promote usability without detracting from the main information.

Accessibility

Accessibility is essential in online design, ensuring that websites are usable by users of all capabilities. This includes considerations for screen readers, structure, screen size, and other features

Screen Readers and Structure: It is vital that screen readers and structures include alternate text and picture titles so that visually impaired civilians can understand the information. Font size and spacing may be modified to accommodate users with visual impairments or other preferences.

W3C Guidelines: Following the W3C accessibility guidelines, which prioritize making content perceivable, operable, understandable, and resilient. This involves providing text alternatives to non-text material, ensuring that all functionality is keyboard accessible, and avoiding content that might cause seizures.

Aesthetics: Colour contrast ensures that colourblind personals can differentiate all content. Proper indentation and font sizes between different sections of the page, such as titles, content, images and footer, make browsing easier.

Screens sizesLastly, it is indispensable to assess the performance of a website across multiple zoom levels and platforms. Consequently, it should be usable and readable at all zoom levels and display sizes.
